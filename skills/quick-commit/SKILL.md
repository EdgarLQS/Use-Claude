## 指令

当用户说以下类似内容时，执行一键提交操作：
- "快速提交"
- "一键提交"
- "quick commit"
- "/quick-commit"
- "提交这个变更"

## 执行流程

### 1. 检查 git 状态
执行 `git status --short` 确认有未提交的变更。
- 如果没有变更，告知用户："工作区是干净的，没有需要提交的变更"
- 如果有变更，继续下一步

### 2. 显示变更摘要
执行 `git diff --stat` 展示变更文件列表。

### 3. 显示完整 diff（详细审查模式）
执行 `git diff` 显示完整的 diff 内容，让用户审查变更详情。

### 4. 确认是否继续
询问用户："是否继续提交？[Y/n]"
- 用户输入 `n` 则取消提交
- 用户输入 `Y` 或直接回车则继续

### 5. 生成提交信息
根据变更内容自动生成建议的提交信息：
- 分析变更类型：feat, fix, refactor, test, docs, perf
- 提取主要变更文件名
- 生成格式：`<type>: <description>`

**变更类型判断规则**：
- 包含 "fix", "bug", "error", "exception", "crash" → `fix`
- 包含 "refactor", "clean", "restructure" → `refactor`
- 包含 "test", "spec" → `test`
- 包含 "doc", "readme", "comment" → `docs`
- 包含 "perf", "optim", "speed" → `perf`
- 默认 → `feat`

### 6. 确认提交信息
显示建议的提交信息，允许用户：
- 直接回车：使用建议的提交信息
- 输入文字：使用自定义提交信息
- 输入 `q`：取消提交

### 7. 添加变更
执行 `git add -A` 添加所有变更到暂存区。

### 8. 创建提交
执行 `git commit -m "<提交信息>"`

**注意**：
- 不要使用 `--no-verify` 跳过钩子
- 不要使用 `--amend` 除非用户明确要求
- 如果 pre-commit 钩子失败，先修复问题再重新提交

### 9. 推送到 GitHub
执行 `git push`

**注意**：
- 不要使用 `--force` 强制推送
- 如果是首次推送，执行 `git push -u origin <branch>`

### 10. 显示提交结果
告知用户：
- ✓ 提交成功
- 提交哈希（短格式）
- 提交信息
- GitHub 提交链接（如果有远程仓库 URL）

## Python 脚本调用

可以使用项目中的 `quick_commit.py` 脚本执行一键提交：

```bash
python quick_commit.py
```

脚本功能：
- 自动检查 git 状态
- 显示变更摘要和完整 diff
- 自动生成提交信息
- 交互式确认
- 自动提交和推送
